# Plan: Issue #1 — M1.0 Setup (Next.js 14, Tailwind, shadcn/ui, Framer Motion)

## Context

GitHub Issue #1 tasks us with scaffolding the frontend foundation for Contracker. The branch to create is `feature/1-setup-nextjs-tailwind-shadcn` (as specified in the issue).

Most of the scaffolding is already done: Next.js 14, Tailwind CSS v3, Framer Motion, all Radix UI packages, and design tokens are all installed and configured. The **two remaining gaps** are:
1. `components/ui/` directory does not exist — the shadcn/ui *component wrapper files* (button.tsx, card.tsx, etc.) need to be created (the Radix UI dependencies are installed but unwrapped)
2. `app/layout.tsx` is a bare shell — needs font loading (Geist Sans, DM Sans) and providers; the `(app)` route group needs a layout with sidebar navigation

## Success Criteria (from issue)
1. Dev server starts cleanly on port 3000
2. Tailwind utilities function properly in components
3. shadcn/ui Button component renders without issues
4. Framer Motion animation executes on load
5. Production build completes successfully

## Step 3 — Update `app/layout.tsx` (root layout)

**Current state:** bare HTML shell, no fonts, no providers.

**Changes:**

- Add `suppressHydrationWarning` on `<html>` (shadcn/ui standard)

**File:** `app/layout.tsx`
